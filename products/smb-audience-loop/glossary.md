# Audience Loop — Glossary

> ALoop-specific terms. For platform structure terms, see [architecture/reference.md](../../architecture/reference.md). For brand-level terms, see [products/icustomer/glossary.md](../icustomer/glossary.md).

| Term | Definition |
|---|---|
| **Master** | The always-on accounts list at the top of Audience Explorer. ABM-style. |
| **Unified Pool** | The always-on contacts pool. Contacts accumulate from all lists + enrichments. |
| **Cohort** | Auto-generated views on top of Master + Unified Pool. Not user-edited. |
| **Suppression** | Rules that exclude records from cohorts (compliance, ad-spend efficiency) |
| **Goal** | Always-on, broad direction. Has a primary KPI. 1 primary + ≤2 secondary per workspace. |
| **Target** | Time-bound, measurable, achievable commitment. E.g., "200 demos in 30 days." |
| **Campaign** | Budgeted, multi-channel, time-bound execution vehicle. NEW first-class app object. |
| **Playbook** | TOFU / MOFU / BOFU template, bound to a target type. |
| **Activation** | The actual push to a platform. Triggers decision trace. NOT attached to a target. |
| **Pulse** | Task-oriented recurring activities (e.g., "HubSpot + LinkedIn audit every Tuesday") |
| **iAgent** | Customer-facing product agent. The 6 ALoop iAgents: Angela, Sam, Lina, Erin, Joel, Brandon. |
