# Agent context — handbooks/

- Audience-specific operational SOPs. Each top-level subfolder (`engineering/`, `product/`, `marketing/`) is owned by that team.
- Update cadence is COLD (quarterly-ish). If a doc here changes weekly, escalate — the process is unstable.
- Cross-reference [architecture/](../architecture/) for system structure; don't duplicate.
- Cross-reference [templates/](../templates/) for templates; don't inline.
- Each subfolder gets its own README.md and skill.md.
- Glossary terms specific to one audience live in `<audience>/glossary.md`. Platform-wide terms live in `architecture/reference.md`.
