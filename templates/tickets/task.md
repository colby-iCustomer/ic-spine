# Task — Jira ticket template

```
Title: [Task Title]
Type: Task
Priority: [Highest | High | Medium | Low]
Epic: [epic key, if applicable]

Description:
## Summary
[Plain-language description of the work item — what and why]

## Current Flow
[As-is state — remove section if not applicable]

## Expected Flow
[To-be state — remove section if not applicable]

## Note
[Additional context, requester]

## Resources
[Links, docs, references — remove if none]

Labels: stream:[prospect|support|dev]
```

## Notes

- Tasks are concrete units of work; if the scope is unclear, file a Spike first.
- Each task should be sizeable to one sprint or less. If bigger, decompose.
