# Story — Jira ticket template

```
Title: [Story Title]
Type: Story
Priority: [Highest | High | Medium | Low]
Epic: [epic key, if applicable]

Description:
## Story
As a [persona], I want [capability], so that [outcome].

### Persona
[Target user roles]

## Acceptance Criteria
- [GIVEN/WHEN/THEN or clear requirement]
- [...]

## Resources
[Links, docs, references — remove if none]

Labels: stream:[prospect|support|dev]
```

## Notes

- Acceptance Criteria use `- ` bullet syntax. Do NOT use `- [ ]` — checkboxes don't render via the Atlassian MCP.
- One persona per story. If you have multiple personas, write multiple stories.
